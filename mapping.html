class Maps.Mapping.holding_register_handling_plc

Base: object

Classe que mapeia os Holding Registers (registradores de escrita) do PLC do Handling.

Os Holding Registers são endereços Modbus que o sistema de controle (máquina principal) utiliza para enviar comandos ao módulo Handling. Cada registrador representa uma ação ou comando específico que o PLC do Handling deve executar.

Este módulo do MPS é responsável por: - Movimentar o braço robótico (garra) - Controlar o magazine alimentador de peças - Gerenciar o painel de sinalizadores (lâmpadas Andon) - Processar botões de controle (Start, Stop, Reset)

Categorias de Registradores:

Controle Remoto e Modo:

  • Habilitar/desabilitar controle remoto

  • Alternar modo automático/manual

Estados da Máquina:

  • Indicar máquina rodando, parada ou em emergência

  • Pulsos de comando (Start, Stop, Reset)

Sinalizadores (Lâmpadas):

  • Painel Andon com cores (verde, amarelo, vermelho)

  • Luz de fundo dos botões

Movimentos do Braço Robótico:

  • Movimentar garra para magazine ou estação

  • Descer/subir garra

  • Abrir/fechar garra

Sistema de Magazine:

  • Avançar/recuar mecanismo de alimentação

Notas Importantes:

  • Todos os valores são booleanos (True = ativar, False = desativar)

  • Os endereços foram escolhidos para deixar espaço para expansões futuras

  • A documentação das lâmpadas está relacionada a padrão Andon (sistema visual de status)

Exemplo de uso:
>>> from Mapping import holding_register_handling_plc as HR
>>> 
>>> # Indicar que máquina está em execução
>>> plc.write_register(HR.MACHINE_RUNNING, True)
>>> 
>>> # Enviar comando para mover garra ao magazine
>>> plc.write_register(HR.GRIPPER_TO_MAGAZINE_ESQ, True)
>>> 
>>> # Acender lâmpada de aviso (amarelo)
>>> plc.write_register(HR.LAMP_YELLOW, True)

  • holding_register_pressing_plc: Comandos para o módulo de prensagem

class Maps.Mapping.holding_register_pressing_plc

Base: object

Classe que mapeia os Holding Registers (registradores de escrita) do PLC de Prensagem.

Os Holding Registers são endereços Modbus que o sistema de controle utiliza para enviar comandos ao módulo Pressing. Cada registrador representa um comando específico para controlar o sistema de prensa e transporte de moedas (peças).

O módulo de Prensagem é responsável por: - Gerenciar a esteira transportadora de peças - Controlar a prensa hidráulica/mecânica - Posicionar moedas (peças) antes da prensagem - Remover moedas prensadas ou defeituosas - Gerenciar sinalizadores de status (lâmpadas)

Categorias de Comandos:

Controle de Esteira e Mecanismos:

  • Liga/desliga esteira de transporte

  • Controla atuadores de bloqueio e avanço

  • Gerencia parada de emergência

Controle de Prensa:

  • Liga prensa (inicia ciclo de prensagem)

  • Monitora sistema de parada/controle

Lógica de Controle:

  • Sinalizadores lógicos (similar ao Andon)

  • Estados de operação (Q1, Q2)

Detalhes dos Registradores:

Esteira e Componentes (0-5):

  • MB_LIGA_ESTEIRA: Energiza motor da esteira

  • MB_REC_BLOQUEADOR: Recua bloqueador (libera peça)

  • MB_REC_COINS: Recua mecanismo de moedas

  • MB_PRESS_ON: Liga prensa

  • MB_REC_ACTUATOR_STOP_COIN: Parada de emergência do atuador

  • MB_AVANCA_COINS: Avança mecanismo de moedas para prensagem

Lógica de Controle (6-9):

  • MB_L_START: Sinal de início (análogo a START do Handling)

  • MB_L_RESET: Sinal de reset (análogo a RESET do Handling)

  • MB_L_Q1, MB_L_Q2: Sinais de estado/qualidade

Notas Importantes:

  • Todos os valores são booleanos (True = ativar, False = desativar)

  • Sequência típica: Liga esteira → Avança moedas → Liga prensa → Aguarda → Recua

  • Sistema de segurança crítico: paradas de emergência devem ser responsivas

  • Operação sincronizada: deve coordenar com sensores do módulo (input_register_pressing_plc)

Exemplo de Ciclo de Prensagem:
>>> from Mapping import holding_register_pressing_plc as HR_Press
>>> 
>>> # Inicia ciclo de prensagem
>>> plc.write_register(HR_Press.MB_LIGA_ESTEIRA, True)  # Esteira se move
>>> time.sleep(0.5)
>>> plc.write_register(HR_Press.MB_AVANCA_COINS, True)  # Posiciona moeda
>>> time.sleep(0.2)
>>> plc.write_register(HR_Press.MB_PRESS_ON, True)      # Liga prensa
>>> time.sleep(2)  # Aguarda prensagem
>>> plc.write_register(HR_Press.MB_PRESS_ON, False)     # Desliga prensa
>>> plc.write_register(HR_Press.MB_REC_COINS, True)     # Recua moedas
>>> # ... continuação do ciclo

  • holding_register_handling_plc: Comandos para módulo Handling

class Maps.Mapping.input_register_handling_plc

Base: object

Classe que mapeia os Input Registers (registradores de leitura) do PLC do Handling.

Os Input Registers são endereços Modbus que o sistema de controle utiliza para ler/monitorar o estado de sensores do módulo Handling. Cada registrador representa a leitura de um sensor ou uma variável de estado específica.

Este módulo fornece feedback sobre: - Posição do braço robótico (sensors de fim de curso) - Presença de peças (sensores de detecção) - Status do magazine - Entrada de comandos do operador (botões) - Lógica de controle do sistema

Tipos de Sensores:

Sensores de Posição do Braço:

Indicam posições-chave durante o ciclo de movimento: - sensor_braco_home: Braço em posição inicial/home - sensor_braco_deixa: Braço deixando o magazine (com peça) - sensor_braco_rejeito: Braço em posição de rejeito

Sensores da Garra:

Indicam status mecânico da garra: - sensor_garra_avancada: Garra avançada (perto da peça) - sensor_garra_recuada: Garra recuada (posição inicial) - sensor_peca_garra: Peça detectada na garra

Sensores do Magazine:

Indicam estado do sistema alimentador: - sensor_magazine_entrada_recuado: Entrada do magazine recuada - sensor_magazine_entrada_avancado: Entrada do magazine avançada

Sensores de Peça e Suporte:

  • sensor_peca_suporte: Peça presente na posição de suporte

Botões de Controle Manua:

Indicam entrada do operador: - button_start: Operador pressionou botão START - button_stop: Operador pressionou botão STOP - button_reset: Operador pressionou botão RESET

Interfaces de Comunicação:

  • MB_IP_FI_CAN: Interface Modbus/CAN (Future CAN integration)

Notas Importantes:

  • Todos os valores são booleanos (True = condição atendida, False = não atendida)

  • Sensores de fim de curso: True quando o braço/garra está na posição indicada

  • Sensores de detecção: True quando objeto/peça é detectado

  • Os botões podem ter lógica de debounce no PLC real

  • Alguns sensores podem ser combinados para detectar múltiplos estados

Exemplo de uso:
>>> from Mapping import input_register_handling_plc as IR
>>> 
>>> # Verificar se braço está em home antes de iniciar
>>> if plc.read_register(IR.sensor_braco_home):
...     plc.write_register(HR.CMD_START, True)
>>> 
>>> # Verificar se há peça na garra
>>> if plc.read_register(IR.sensor_peca_garra):
...     # Mover para estação
...     plc.write_register(HR.GRIPPER_TO_STATION_DIR, True)
>>> 
>>> # Esperar até garra estar totalmente recuada
>>> while not plc.read_register(IR.sensor_garra_recuada):
...     time.sleep(0.1)

class Maps.Mapping.input_register_pressing_plc

Base: object

Classe que mapeia os Input Registers (registradores de leitura) do PLC de Prensagem.

Os Input Registers são endereços Modbus que o sistema de controle utiliza para ler/monitorar o estado de sensores e variáveis do módulo Pressing. Cada registrador fornece feedback crítico para tomada de decisão e sincronização.

Este módulo fornece feedback sobre: - Botões de controle do operador - Posição de peças no sistema - Status de sensores de bloqueio e barreira - Sinais de posicionamento de moedas

Categorias de Sensores:

Controles Manuais (0-2):

  • Botões do operador: START, STOP, RESET

  • Permitem comando manual do sistema

Sensores de Detecção de Peças (3-5):

  • Detecção de peças avançadas na esteira

  • Detecção em diferentes pontos do ciclo

Sensores de Posicionamento (6-9):

  • Sensores de fim de curso

  • Posições críticas para sincronização

Detalhes dos Registradores:

Botões Manuais (0-2):

  • MB_START: Operador pressiona botão START

  • MB_STOP: Operador pressiona botão STOP

  • MB_RESET: Operador pressiona botão RESET

Sensores de Peça (3-5):

  • MB_PART_AV: Peça avançada (presença na área de prensagem)

  • MB_PC_COIN: Peça (moeda) presente no sistema

  • MB_PC_FIM: Peça no fim de curso (ready para prensagem)

Sensores de Barreira/Bloqueio (6-7):

  • MB_SENSOR_IND: Sensor indicador (detector indutor)

  • MB_BARREIRA_IND: Barreira indutiva (detecção de presença)

Posições de Bloqueio e Moedas (8-10):

  • MB_BLOQ_FRONT: Bloqueador em posição frontal

  • MB_COIN_REC: Moedas em posição recuada (home)

  • MB_COIN_FRONT: Moedas em posição frontal (sob prensa)

Fluxo Típico de Leitura:

  1. Verificar MB_START ou automático

  2. Liga esteira (MB_LIGA_ESTEIRA)

  3. Aguarda MB_PC_FIM (peça em posição)

  4. Liga prensa (MB_PRESS_ON)

  5. Verifica conclusão

  6. Retorna à posição inicial

Notas Importantes:

  • Todos os valores são booleanos (True = condição detectada, False = não)

  • Ordem crítica: antes de prensagem, peça DEVE estar em MB_PC_FIM

  • Sensores indutivos (MB_SENSOR_IND, MB_BARREIRA_IND): não mecânicos, confiaáveis

  • Lógica de sincronização: ler sensores antes de enviar próximo comando

  • Tempo de resposta: cada sensor tem atraso mecânico (bufferização pode ser necessária)

Exemplo de Lógica de Segurança:
>>> from Mapping import input_register_pressing_plc as IR_Press
>>> from Mapping import holding_register_pressing_plc as HR_Press
>>> 
>>> # Segurança: apenas ativar prensa se moeda está em posição frontal
>>> if plc.read_register(IR_Press.MB_COIN_FRONT):
...     plc.write_register(HR_Press.MB_PRESS_ON, True)
... else:
...     print("ERRO: Moeda não está em posição de prensagem!")
...     # Aciona alarme, registra erro
