Digital Twin

Creating a Digital Twin

class Server.DigitalTwin.DigitalTwin

Classe principal que representa o Digital Twin do Sistema MPS Festo.

O Digital Twin é uma representação digital do sistema físico que:

  • Sincroniza estado: Mantém sincronizados todos os sensores, atuadores e estados de controle através de um servidor Modbus TCP.

  • Permite monitoramento: Permite visualizar e monitorar o estado em tempo real através de softwares como FactoryIO.

  • Permite controle remoto: Permite que softwares externos (clientes Modbus) enviem comandos e parâmetros de controle.

  • Gerencia comunicação: Utiliza protocolo Modbus TCP (porta 502 padrão) para comunicação com outros componentes do sistema.

O servidor Modbus mantém dois tipos principais de dados:

  • Discrete Inputs (DI): Valores booleanos para sensores e atuadores digitais

  • Holding Registers (HR): Valores numéricos para posições, velocidades e setpoints

Atributos:

logger (LoggerManager): Sistema de logging para rastrear eventos e erros.

server (ModbusServer): Instância do servidor Modbus TCP que gerencia comunicação.

db (DataBank): Banco de dados do servidor Modbus onde os estados são armazenados.

DI (type): Referência à enumeração DI para uso como self.DI.LAMP_GREEN_DT.

INPUT_HR (type): Referência à enumeração INPUT_HR para uso como self.INPUT_HR.Crane_Fedder_Setpoint_X.

Exemplo de uso completo:
>>> # Criar instância do Digital Twin
>>> dt = DigitalTwin()
>>> 
>>> # Acionar cilindro do alimentador
>>> dt.set_parameter(DI.Cylinder_Pusher_Feeder, True)
>>> 
>>> # Definir posição do braço robótico (X=5.0, Z=3.0 no FactoryIO)
>>> dt.set_parameter(INPUT_HR.Crane_Fedder_Setpoint_X, 500)
>>> dt.set_parameter(INPUT_HR.Crane_Fedder_Setpoint_Z, 300)
>>> 
>>> # Sincronizar todos os dados com servidor Modbus
>>> dt.commit_all()
>>> 
>>> # Agora clientes Modbus (ex: FactoryIO) podem ler esses valores
Notas técnicas:

  • O servidor Modbus inicia em modo não-bloqueante (no_block=True) para não bloquear a execução do programa principal.

  • A porta 502 é a porta padrão do Modbus TCP em sistemas Unix/Linux. Em Windows, pode-se usar porta 502 se executar com privilégios de administrador, ou uma porta alternativa como 5020.

  • Erros na inicialização são registrados no log mas não interrompem a execução.

commit_all()

Sincroniza todos os Discrete Inputs e Holding Registers com o servidor Modbus.

Este método é crucial para que as alterações feitas via set_parameter() se tornem visíveis para clientes Modbus (como FactoryIO). Ele:

  1. Envia Discrete Inputs: Transfere o array di (valores booleanos) para o servidor Modbus, começando no endereço 0.

  2. Envia Holding Registers: Transfere o array input_hr (valores numéricos) para o servidor Modbus, começando no endereço 0.

  3. Trata compatibilidade: Verifica qual método de escrita o servidor Modbus suporta (set_input_registers ou set_words) e usa o apropriado.

  4. Registra operações: Registra cada operação bem-sucedida no log para rastreamento de eventos.

Fluxo típico de uso:

  1. Chamar set_parameter() múltiplas vezes para alterar valores locais

  2. Chamar commit_all() uma vez para sincronizar tudo de uma vez

  3. Clientes Modbus agora conseguem ler os novos valores

Notas importantes:

  • Chamar commit_all() é obrigatório para que alterações se tornem visíveis a clientes Modbus. Caso contrário, o servidor terá dados desatualizados.

  • Em operações críticas, você pode chamar commit_all() a cada set_parameter(), mas é menos eficiente.

  • Os comentários no código (“Fazendo o set de todos…”) indicam que há redundância de dados (INPUT_HR vs Words) que pode ser otimizada.

Exemplos:
>>> dt = DigitalTwin()
>>> # Configurar vários parâmetros
>>> dt.set_parameter(DI.LAMP_GREEN_DT, True)
>>> dt.set_parameter(DI.LAMP_RED_DT, False)
>>> dt.set_parameter(INPUT_HR.Crane_Fedder_Setpoint_X, 500)
>>> # Sincronizar uma única vez
>>> dt.commit_all()
Ver também:

  • set_parameter(): Para alterar valores individuais antes de commit

set_parameter(parameter, value)

Define o valor de um parâmetro específico (Discrete Input ou Holding Register).

Este método permite modificar o estado de um único sensor, atuador ou parâmetro de controle. Os valores são armazenados localmente no array global (di ou input_hr) e só se tornam visíveis aos clientes Modbus após chamar commit_all().

Args:
parameter (DI | INPUT_HR): Identificador do parâmetro a ser modificado.

Deve ser um membro da enumeração DI ou INPUT_HR. Exemplo: DI.LAMP_GREEN_DT ou INPUT_HR.Crane_Fedder_Setpoint_X

value (int | bool | float): Novo valor para o parâmetro.

  • Para parâmetros DI: deve ser booleano (True/False)

  • Para parâmetros INPUT_HR: deve ser inteiro (0-1000)

  • Valores de outros tipos serão convertidos automaticamente

Raises:

TypeError: Se o parâmetro não for instância de DI nem INPUT_HR, registra erro no log.

Notas importantes:

  • Conversão automática: Booleans são convertidos para True/False para DI, números são convertidos para inteiros para INPUT_HR.

  • Escala de INPUT_HR: Valores de 0 a 1000 mapeiam para 0 a 10.0 no FactoryIO. Multiplicar por 100: valor_factoryio = valor_input_hr / 100.0

  • Sincronização necessária: Após modificar parâmetros, você DEVE chamar commit_all() para que as mudanças sejam enviadas ao servidor Modbus.

  • Sem verificação de limites: O método não valida se o valor está no intervalo esperado. Cabe ao código chamador garantir valores válidos.

Exemplos de uso:
>>> dt = DigitalTwin()
>>> 
>>> # Exemplo 1: Controlar lâmpadas (Discrete Inputs)
>>> dt.set_parameter(DI.LAMP_GREEN_DT, True)   # Acende lâmpada verde
>>> dt.set_parameter(DI.LAMP_RED_DT, False)    # Apaga lâmpada vermelha
>>> 
>>> # Exemplo 2: Controlar atuadores (Discrete Inputs)
>>> dt.set_parameter(DI.Cylinder_Pusher_Feeder, True)  # Ativa cilindro
>>> dt.set_parameter(DI.Conveyor_Job, True)    # Inicia esteira
>>> 
>>> # Exemplo 3: Definir posições (Holding Registers)
>>> dt.set_parameter(INPUT_HR.Crane_Fedder_Setpoint_X, 500)  # X = 5.0
>>> dt.set_parameter(INPUT_HR.Crane_Fedder_Setpoint_Z, 300)  # Z = 3.0
>>> 
>>> # IMPORTANTE: Sincronizar com servidor
>>> dt.commit_all()
Ver também:

  • DI: Enumeração de Discrete Inputs

  • INPUT_HR: Enumeração de Holding Registers

  • commit_all(): Para sincronizar parâmetros com servidor Modbus
